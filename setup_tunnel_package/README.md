# setup_tunnel.sh

Cloudflare Tunnel bootstrap utility.

This script installs `cloudflared` (if missing) and creates a global helper command:

```text
start-tunnel
```

The helper launches a Cloudflare tunnel pointing to:

```text
http://localhost:80
```

## Files

- `setup_tunnel.sh` — Cloudflare tunnel setup utility

## Usage

Make executable:

```bash
chmod +x setup_tunnel.sh
```

Run:

```bash
./setup_tunnel.sh
```

## What the script does

### 1. Checks for cloudflared

Verifies whether:

```bash
cloudflared
```

already exists on the system.

### 2. Installs cloudflared if needed

Downloads the latest AMD64 Debian package from GitHub:

```text
https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
```

Then installs it with:

```bash
sudo dpkg -i
```

### 3. Creates global shortcut

Writes a helper script to:

```text
/usr/local/bin/start-tunnel
```

The helper runs:

```bash
cloudflared tunnel --url http://localhost:80
```

### 4. Makes helper executable

Runs:

```bash
sudo chmod +x /usr/local/bin/start-tunnel
```

## After setup

From any directory:

```bash
start-tunnel
```

This launches a temporary Cloudflare tunnel to the local web server running on port 80.

## Notes

- The outer heredoc wrapper was removed from the packaged script.
- The actual script starts directly with:
  ```bash
  #!/usr/bin/env bash
  ```
- Requires:
  - sudo privileges
  - Debian/Ubuntu-compatible `.deb` support
  - AMD64 architecture
- Tunnel URLs generated by `cloudflared tunnel --url` are typically temporary unless authenticated tunnels are configured.
